## **🔹 Flask & Backend Questions**
### 4️⃣ **How does Flask handle routing in your project?**  
✅ **Answer:**  
Flask uses decorators like `@app.route("/")` to define URL routes.  
For example, the home page route:  
```python
@app.route("/")
def home():
    return render_template("index.html")  
```
Each route is mapped to a function that either **renders an HTML template** or **processes user input**.

---

### 5️⃣ **How did you implement user authentication in Flask?**  
✅ **Answer:**  
I used Flask **sessions** for user authentication. When a user logs in, their session stores their `user_id`:  
```python
@app.route("/login", methods=["POST"])
def login():
    session["user_id"] = user[0]  # Stores user ID in session
```
For admin authentication, I used **hardcoded credentials** for now but can integrate it with a database later.

---

### 6️⃣ **How does Flask handle sessions, and how are they used in your project?**  
✅ **Answer:**  
Flask **sessions** store temporary user data using cookies. I use them to keep users logged in:  
```python
session["user_id"] = user_id  # Stores logged-in user ID
```
To log out, I **clear the session**:  
```python
session.clear()  
```
This ensures secure authentication.

### 7️⃣ **How did you manage user roles (admin vs. user) in your application?**  
✅ **Answer:**  
- **Users** can **sign up, log in, submit complaints, and track their submissions**.  
- **Admins** log in separately and have **access to complaint categorization, user management, and resolution tracking**.  

I use Flask sessions to differentiate roles:  
```python
if "admin" in session:
    return redirect(url_for("admin_dashboard"))
else:
    return redirect(url_for("login"))
```

### 8️⃣ **How does your project store complaints, and how do you retrieve them?**  
✅ **Answer:**  
Complaints are stored in an **SQLite database** with columns like `text`, `category`, `user_id`, `submission_date`.  
To retrieve complaints:  
```python
cursor.execute("SELECT * FROM complaints WHERE user_id = ?", (session["user_id"],))
```

This fetches **only complaints related to the logged-in user**.

## **🔹 NLP & Machine Learning (BERT Model)**
### 1️⃣0️⃣ **Why did you choose BERT for complaint classification instead of other models?**  
✅ **Answer:**  
BERT (Bidirectional Encoder Representations from Transformers) is **state-of-the-art for NLP** and understands **context better than traditional models** like TF-IDF or Naïve Bayes. It helps classify complaints accurately even with **complex sentence structures**.

---

### 1️⃣1️⃣ **How does BERT process and classify text in your project?**  
✅ **Answer:**  
- **Tokenization:** Converts text into numerical tokens using `BertTokenizer`.  
- **Embedding & Attention:** BERT analyzes word relationships in context.  
- **Prediction:** Outputs a category label.  

Example classification function:  
```python
inputs = tokenizer(text, return_tensors="pt", truncation=True, padding="max_length", max_length=256)
outputs = model(**inputs)
category_id = torch.argmax(outputs.logits, dim=1).item()
return category_mapping.get(category_id, "Unknown")
```

---

### 1️⃣2️⃣ **What preprocessing steps did you apply to the complaint text before passing it to BERT?**  
✅ **Answer:**  
1. **Lowercasing** the text  
2. **Removing special characters** (e.g., `@`, `#`, punctuation)  
3. **Tokenization** using `BertTokenizer`  
4. **Padding & truncation** for uniform input length  

---

If a complaint doesn’t match an existing category, the model returns `"Unknown"`, and the admin can manually review it.
